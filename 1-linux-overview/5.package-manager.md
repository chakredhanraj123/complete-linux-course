# 05 - Package Managers in Linux

# 📦 What is a Package?

A **package** is a compressed file that contains everything required to install an application on a Linux system.

A package usually contains:

- Application binaries
- Configuration files
- Libraries
- Documentation
- Installation scripts
- Metadata

Think of a package as an installer (`.exe`) on Windows. Instead of downloading software from different websites, Linux distributes software through packages.

### Examples

- nginx.deb
- docker.rpm
- kubectl.deb
- terraform.rpm

---

# 🛠 What is a Package Manager?

A **Package Manager** is a software tool that automates the installation, updating, configuration, and removal of software packages.

Instead of manually downloading applications and their dependencies, a package manager performs all tasks automatically.

A package manager can:

- Install software
- Update package lists
- Upgrade installed software
- Remove software
- Resolve dependencies
- Verify package authenticity
- Download packages from repositories
- Maintain a local package database

Without a package manager, installing software on Linux would be much more difficult.

---

# ❓ Why Do We Need a Package Manager?

Imagine installing Docker manually.

Without a package manager, you would need to:

- Search for Docker on the internet
- Download the correct version
- Download all required libraries
- Resolve dependencies
- Configure everything manually
- Handle compatibility issues
- Update the software manually

With a package manager, all of this is done using a single command.

```bash
sudo apt install docker.io
```

Benefits include:

- Faster installation
- Automatic dependency management
- Secure installation
- Easy updates
- Easy removal
- Version management
- Consistent software installation

---

# 🌐 What is a Repository?

A **repository** is an online server that stores Linux software packages.

You can think of a repository as the Linux equivalent of the Google Play Store or Apple App Store.

Repositories contain thousands of packages such as:

- Docker
- Git
- Nginx
- Apache
- Python
- Node.js
- Terraform
- Kubernetes CLI (kubectl)

Package managers download software directly from these repositories.

---

# 📂 How Does a Package Manager Work?

When you execute:

```bash
sudo apt install nginx
```

The package manager performs the following steps:

1. Checks the configured repositories.
2. Searches for the requested package.
3. Downloads the package.
4. Downloads all required dependencies.
5. Verifies the package signature.
6. Installs the software.
7. Updates the local package database.
8. Configures the software automatically.

Everything happens automatically.

---

# 🔗 What are Dependencies?

Many applications require additional software packages to function properly.

These required packages are called **dependencies**.

For example, installing Git may also install:

- libcurl
- openssl
- perl
- gettext

Package managers automatically install these dependencies.

# 📁 Ubuntu Repository Configuration

Ubuntu stores repository information in:

```text
/etc/apt/sources.list
```

Additional repository files are stored in:

```text
/etc/apt/sources.list.d/
```

Example repository entry:

```text
deb http://archive.ubuntu.com/ubuntu noble main restricted universe multiverse
```

---

# 🔄 Why Run `apt update`?

After installing Ubuntu, the software packages available on the installation media may be outdated.

Running:

```bash
sudo apt update
```

downloads the latest package information from the configured repositories.

**Important:** `apt update` does **not** upgrade installed software.

It only refreshes the package list.

---

# ⬆️ What Does `apt upgrade` Do?

Once the package list is updated, run:

```bash
sudo apt upgrade -y
```

This installs newer versions of all installed packages.

It also applies important bug fixes and security updates.

---

# 📊 Difference Between `apt update` and `apt upgrade`

| Command | Purpose |
|----------|----------|
| `sudo apt update` | Refresh package information from repositories |
| `sudo apt upgrade` | Upgrade installed packages |

Always execute:

```bash
sudo apt update
sudo apt upgrade -y
```

# 💾 Installing Local Packages

Sometimes packages are downloaded manually instead of from repositories.

### Ubuntu

```bash
sudo dpkg -i package.deb
```

If dependencies are missing:

```bash
sudo apt install -f
```

### RHEL / Fedora

```bash
sudo rpm -ivh package.rpm
```

---

# 🔒 Package Verification

Package managers verify:

- Digital signatures
- Package integrity
- Checksums

This ensures that downloaded packages have not been modified or tampered with.

---

# 🚀 Best Practices

- Always update package lists before installing software.

```bash
sudo apt update
```

- Upgrade packages regularly.

```bash
sudo apt upgrade -y
```

- Remove unused dependencies.

```bash
sudo apt autoremove
```

- Use official repositories whenever possible.

- Avoid downloading random `.deb` or `.rpm` packages from untrusted websites.

# 04 - Setup Linux Environment (Windows, macOS & Linux)

## Overview

To practice Linux commands, you need a Linux environment. Instead of installing Linux directly on your machine, you can run Linux inside a **Docker container**. This approach is lightweight, fast, portable, and works on **Windows, macOS, and Linux**.

> **Recommended:** Docker Desktop + Ubuntu Container

---

## Available Options

There are multiple ways to run Linux:

- Cloud Virtual Machine (AWS, Azure, GCP)
- WSL2 (Windows Subsystem for Linux)
- VirtualBox
- VMware
- HyperKit (macOS)
- **Docker Container (Recommended ✅)**

### Why Docker?

- Lightweight and fast
- Free to use
- No dual boot or virtual machine required
- Same setup across Windows, macOS, and Linux
- Easy to create, delete, and recreate environments

---

# Prerequisites

- Install **Docker Desktop**
- Ensure Docker is running
- Open **PowerShell** (Windows) or **Terminal** (macOS/Linux)

Verify Docker installation:

```bash
docker --version
```

---

# Create a Persistent Data Folder

### Windows

Create the following directory:

```text
C:\Users\<YOUR_USERNAME>\Downloads\ubuntu-data
```

### macOS / Linux

```bash
mkdir -p ~/ubuntu-data
```

---

# Run Ubuntu Container

## Windows (PowerShell)

```powershell
docker run -dit `
  --name ubuntu-container `
  --hostname ubuntu-dev `
  --restart unless-stopped `
  --cpus="2" `
  --memory="4g" `
  --mount type=bind,source="C:/Users/<YOUR_USERNAME>/Downloads/ubuntu-data",target=/data `
  -v /var/run/docker.sock:/var/run/docker.sock `
  -p 2222:22 `
  -p 8080:80 `
  --env TZ=Asia/Kolkata `
  --env LANG=en_US.UTF-8 `
  ubuntu:latest /bin/bash
```

> Replace `<YOUR_USERNAME>` with your Windows username.

---

## macOS / Linux

```bash
docker run -dit \
  --name ubuntu-container \
  --hostname ubuntu-dev \
  --restart unless-stopped \
  --cpus="2" \
  --memory="4g" \
  --mount type=bind,source=$HOME/ubuntu-data,target=/data \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -p 2222:22 \
  --env TZ=Asia/Kolkata \
  --env LANG=en_US.UTF-8 \
  ubuntu:latest /bin/bash
```

# Access the Container

```bash
docker exec -it ubuntu-container bash
```

Verify Ubuntu:

```bash
cat /etc/os-release
```

Check current user:

```bash

```

# Useful Docker Commands

| Command | Description |
|----------|-------------|
| `docker ps` | List running containers |
| `docker ps -a` | List all containers |
| `docker start ubuntu-container` | Start the container |
| `docker stop ubuntu-container` | Stop the container |
| `docker restart ubuntu-container` | Restart the container |
| `docker exec -it ubuntu-container bash` | Open a shell inside the container |
| `docker rm -f ubuntu-container` | Remove the container |

---

# Docker Run Parameters

| Parameter | Description |
|-----------|-------------|
| `-d` | Run container in detached mode |
| `-i` | Keep STDIN open |
| `-t` | Allocate a terminal |
| `--name` | Assign a container name |
| `--hostname` | Set the container hostname |
| `--restart unless-stopped` | Restart automatically after system reboot |
| `--cpus="2"` | Allocate 2 CPU cores |
| `--memory="4g"` | Allocate 4 GB RAM |
| `--mount` | Mount a local folder for persistent storage |
| `-v /var/run/docker.sock:/var/run/docker.sock` | Allow Docker commands inside the container (optional) |
| `-p 2222:22` | Map SSH port |
| `-p 8080:80` | Map HTTP port |
| `--env TZ=Asia/Kolkata` | Set timezone |
| `--env LANG=en_US.UTF-8` | Set language |
| `ubuntu:latest` | Ubuntu Docker image |
| `/bin/bash` | Start the Bash shell |

---

# Why Use a Bind Mount?

The bind mount stores your files on the host machine instead of inside the container.

```
Host Machine
└── ubuntu-data
      │
      ▼
Container
└── /data
```

Even if the container is deleted, your files remain safe.

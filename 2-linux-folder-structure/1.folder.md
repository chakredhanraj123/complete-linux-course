# Understanding the Linux Folder Structure

Linux organizes files and directories in a **hierarchical structure** that starts from the **root directory (`/`)**. Every file, application, configuration, and device is located under this single directory. Understanding the Linux filesystem is essential for Linux administration, DevOps, and system troubleshooting.

---

# Symbolic Links (Modern Linux)

Modern Linux distributions use symbolic links to simplify the filesystem layout. These directories point to their counterparts under the `/usr` directory.

| Directory | Description |
|-----------|-------------|
| `/bin -> /usr/bin` | Essential user commands and binaries. |
| `/sbin -> /usr/sbin` | Administrative system binaries for root users. |
| `/lib -> /usr/lib` | Shared libraries and kernel modules. |

> **Note:** These are symbolic links, not separate directories.

---

# Important System Directories

| Directory | Description |
|-----------|-------------|
| `/boot` | Stores bootloader files, Linux kernel, and files required during system startup. Usually not present inside containers. |
| `/usr` | Contains most installed applications, libraries, documentation, and utilities. |
| `/var` | Stores variable data such as logs, cache files, mail queues, and temporary application data. |
| `/etc` | Stores system-wide configuration files for applications and services. |

---

# User & Application Directories

| Directory | Description |
|-----------|-------------|
| `/home` | Default location for regular users' home directories. |
| `/opt` | Used for installing optional or third-party software packages. |
| `/srv` | Stores data served by services such as web or FTP servers. |
| `/root` | Home directory of the root (administrator) user. |

---

# Temporary & Virtual Directories

| Directory | Description |
|-----------|-------------|
| `/tmp` | Stores temporary files. Contents are typically removed after reboot. |
| `/run` | Stores runtime information such as process IDs (PID), sockets, and lock files. |
| `/proc` | Virtual filesystem that provides information about running processes and the Linux kernel. |
| `/sys` | Virtual filesystem containing hardware and kernel-related information. |
| `/dev` | Contains device files representing hardware devices (e.g., disks, terminals, and special devices like `/dev/null`). |

---

# Mount Points

| Directory | Description |
|-----------|-------------|
| `/mnt` | Temporary mount point for manually mounted filesystems. |
| `/media` | Automatically mounts removable devices such as USB drives and CDs/DVDs. |
| `/data` | Custom directory commonly used for storing application data, backups, or mounted storage (not part of the official Linux filesystem standard). |

---

# Quick Summary

- `/etc` → System configuration files
- `/usr` → Applications and libraries
- `/var` → Logs and variable data
- `/home` → User home directories
- `/root` → Root user's home directory
- `/tmp` → Temporary files
- `/proc` → Process and kernel information
- `/sys` → Hardware and kernel information
- `/dev` → Device files
- `/mnt` → Manual mount point
- `/media` → Removable media
- `/opt` → Third-party software
- `/boot` → Boot files
- `/run` → Runtime process information
- `/srv` → Service-related data

---

## Key Takeaways

- Linux follows a **single hierarchical filesystem** that starts from the root directory (`/`).
- Configuration files are stored in **`/etc`**, logs in **`/var`**, applications in **`/usr`**, and user data in **`/home`**.
- Virtual filesystems like **`/proc`** and **`/sys`** provide real-time system and hardware information.
- Understanding these directories is essential for Linux administration, troubleshooting, Docker, Kubernetes, and DevOps.
